Lab 6: Build a Full-Stack App with Agent Mode

Use Copilot's Agent Mode to build a complete todo application with a React frontend and Express backend. Experience autonomous multi-file creation, terminal commands, and iterative problem solving.

🎯 Objectives

  • Activate and use Agent Mode in VS Code
  • Provide a natural language specification for a full-stack app
  • Observe Agent Mode creating multiple files autonomously
  • Review and approve terminal commands (npm install, etc.)
  • Watch Agent Mode iteratively fix errors

📋 Prerequisites

  • Node.js 18+ and npm installed
  • VS Code 1.99+ with GitHub Copilot
  • Copilot Pro, Pro+, or Business subscription (Agent Mode required)
  • Completed Module 6 on Agent Mode

📝 Step-by-Step Instructions

1Set Up an Empty Project
  1. Create a new folder: copilot-todo-app
  2. Open it in VS Code
  3. Initialize git: open terminal and run git init
  4. This gives Agent Mode a clean workspace to build in
2Activate Agent Mode
  1. Open Copilot Chat (Ctrl+Shift+I)
  2. At the top of the chat panel, find the mode dropdown (shows "Ask" by default)
  3. Click the dropdown and select "Agent"
  4. You'll see the chat icon change to indicate Agent Mode is active
ℹ️
In Agent Mode, Copilot can create files, edit files, run terminal commands, and iterate on errors — all autonomously. You'll be asked to approve each terminal command before it runs.
3Provide the Application Specification

Paste this complete specification into the Agent Mode chat:

Agent Mode Prompt — paste this entire block
Build a full-stack Todo application with the following specs:

## Backend (Express.js)
- File: server/index.js
- Express server on port 3001
- In-memory storage (array of todo objects)
- REST API endpoints:
  - GET /api/todos - List all todos (support ?status=completed|active filter)
  - POST /api/todos - Create todo { title: string, description?: string }
  - PUT /api/todos/:id - Update todo (toggle completed, edit title/description)
  - DELETE /api/todos/:id - Delete a todo
  - DELETE /api/todos - Delete all completed todos (bulk cleanup)
- CORS enabled for localhost:3000
- Each todo: { id: uuid, title, description, completed: false, createdAt, updatedAt }
- Proper error handling and validation

## Frontend (React with Vite)
- Create React app in client/ folder using Vite
- Components:
  - App.jsx - Main layout
  - TodoList.jsx - Renders filtered list of todos
  - TodoItem.jsx - Single todo with checkbox, edit, delete
  - AddTodo.jsx - Form to create new todo (title + optional description)
  - FilterBar.jsx - Filter buttons: All, Active, Completed + item count
- Features:
  - Add new todos with Enter key
  - Toggle todo completion with checkbox
  - Inline editing on double-click
  - Delete individual todos
  - "Clear completed" button
  - Filter by All/Active/Completed
  - Show count: "X items left"
- Styling: Modern CSS with CSS modules or a single styles.css
  - Clean, minimal design with good spacing
  - Smooth transitions for checkbox and delete
  - Responsive layout

## Setup
- Create package.json files for both server and client
- Add a root package.json with scripts to run both concurrently
- Install all necessary dependencies

Press Enter and watch Agent Mode work. It will:

  1. Create the folder structure (server/, client/)
  2. Generate all source files
  3. Ask permission to run terminal commands (npm init, npm install, etc.)
  4. If it encounters errors, it will read them and fix the issues automatically
4Review and Approve Actions

As Agent Mode works, you'll see:

  • File creation notifications — Review each file diff before accepting
  • Terminal command prompts — You must click "Run" to approve commands like npm install
  • Iterative fixes — If a build fails, Agent Mode reads the error and attempts to fix it
⚠️
Always read terminal commands before approving! Legitimate commands will be things like npm install express cors uuid, npm create vite@latest, and npm run dev. Be cautious of any command that seems unusual.
5Test the Application
  1. Once Agent Mode finishes, start both servers (it may have already done this)
  2. If not, run: npm run dev from the root (or the command Agent Mode created)
  3. Open http://localhost:3000 in your browser
  4. Test all features:
  • ✅ Add a todo with just a title
  • ✅ Add a todo with title + description
  • ✅ Check/uncheck to toggle completion
  • ✅ Double-click to inline edit
  • ✅ Delete a single todo
  • ✅ Filter by All / Active / Completed
  • ✅ "Clear completed" removes all done items
  • ✅ Item count updates correctly
✅ Expected Result
A fully working todo app in the browser with all CRUD operations functional, real-time UI updates, and a clean modern design. The backend API responds correctly to all endpoint tests.
6Iterate with Agent Mode

If anything isn't working perfectly, continue the conversation:

Follow-up prompts to try
// If styling needs work:
"The styling looks basic. Make it more polished with shadows, transitions, and better spacing."

// If a feature is missing:
"The inline editing on double-click isn't working. Please fix it."

// To add features:
"Add a dark/light theme toggle button to the header."

// To add persistence:
"Add localStorage persistence so todos survive page refresh."

💡 Tips & Best Practices

💡
  • • Detailed specs = better results. The more specific your initial prompt, the fewer iterations needed.
  • • Let it finish. Don't interrupt Agent Mode mid-flow — let it complete its plan.
  • • Use git. Commit before Agent Mode runs so you can revert if needed: git add . && git commit -m "before agent mode"
  • • Check the file tree. Agent Mode shows which files it created/modified — review the structure.
