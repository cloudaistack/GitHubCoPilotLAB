Lab 9: Custom Instructions Lab

Configure Copilot for a team project with custom instructions, reusable prompt templates, and validation that the AI follows your coding standards consistently.

🎯 Objectives

  • Create a .github/copilot-instructions.md file
  • Write reusable prompt templates in .github/prompts/
  • Validate Copilot follows your custom instructions
  • Understand how instructions affect code generation

📝 Step-by-Step Instructions

1Create the Project Structure
  1. Create a folder team-project and open it in VS Code
  2. Initialize: npm init -y and install TypeScript
  3. Create the folder structure: .github/, .github/prompts/, src/
2Write Custom Instructions

Create .github/copilot-instructions.md with these team standards:

.github/copilot-instructions.md
# Copilot Instructions for Team Project

## Language & Framework
- Use TypeScript with strict mode enabled
- Use functional components with React hooks (no class components)
- Use Zod for runtime validation, never manual type checking
- Use date-fns for date manipulation, never raw Date methods

## Code Style
- Prefer `const` over `let`. Never use `var`
- Use arrow functions for callbacks and small utilities
- Use named exports, not default exports
- Maximum function length: 30 lines. Extract helpers if longer
- All functions must have explicit return types

## Naming Conventions
- Files: kebab-case (e.g., `user-service.ts`)
- Functions/variables: camelCase
- Types/Interfaces: PascalCase, prefix interfaces with 'I' (e.g., `IUserProfile`)
- Constants: UPPER_SNAKE_CASE
- Boolean variables: prefix with is/has/should (e.g., `isActive`)

## Error Handling
- Always use custom error classes extending BaseError
- Never throw plain strings or generic Error
- All async functions must have try/catch with proper error logging
- Use Result pattern (Ok/Err) for expected failure cases

## Testing
- Use Vitest for testing
- Test file naming: `*.test.ts` co-located with source
- Minimum 80% branch coverage
- Use factory functions for test data, not inline objects

## Documentation
- All public functions must have JSDoc with @param, @returns, @throws
- Include usage examples in JSDoc for complex functions

## Security
- Never log sensitive data (passwords, tokens, PII)
- Always validate and sanitize user input
- Use parameterized queries for database operations
3Create Prompt Templates

Create .github/prompts/create-api-endpoint.prompt.md:

.github/prompts/create-api-endpoint.prompt.md
# Create API Endpoint

Create a new API endpoint with the following specification:

## Endpoint: {{METHOD}} {{PATH}}

### Requirements:
- Create the route handler in `src/routes/`
- Create the service function in `src/services/`
- Create Zod validation schema in `src/schemas/`
- Follow team coding standards from copilot-instructions.md
- Include error handling with custom error classes
- Add JSDoc documentation
- Create a test file with at least 5 test cases

### Input: {{INPUT_SCHEMA}}
### Output: {{OUTPUT_SCHEMA}}
### Business Logic: {{DESCRIPTION}}

Create another template .github/prompts/create-data-model.prompt.md:

.github/prompts/create-data-model.prompt.md
# Create Data Model

Create a TypeScript data model with the following:

## Model: {{MODEL_NAME}}

### Fields: {{FIELDS}}

### Requirements:
- TypeScript interface with 'I' prefix
- Zod validation schema
- Factory function for creating instances with defaults
- Serialization/deserialization methods
- Test factory function for generating test data
- Follow all team coding standards
4Test the Instructions

Create src/services/user-service.ts and ask Copilot to generate a user service. Verify it follows your custom instructions:

  • ✅ Uses TypeScript strict types?
  • ✅ Named exports (not default)?
  • ✅ Arrow functions for callbacks?
  • ✅ Explicit return types?
  • ✅ Custom error classes?
  • ✅ JSDoc on all public functions?
  • ✅ Boolean vars prefixed with is/has?
  • ✅ Zod for validation (not manual)?
✅ Expected
Copilot should generate code that follows most or all of your custom instructions. If it misses some, refine your instructions file and try again.
5Use Prompt Templates

In Copilot Chat, reference your prompt template:

Copilot Chat
Use #prompt:create-api-endpoint with:
- METHOD: POST
- PATH: /api/users
- INPUT_SCHEMA: { name: string, email: string, role: 'admin' | 'user' }
- OUTPUT_SCHEMA: { id: string, name: string, email: string, role: string, createdAt: string }
- DESCRIPTION: Create a new user with email uniqueness validation

Copilot should generate all the files specified in the template: route handler, service, schema, and tests.
