🗂️ The Vulnerable Code

Create vulnerable_app.py with this intentionally insecure code:

vulnerable_app.py — Paste this
from flask import Flask, request, render_template_string
import sqlite3
import hashlib
import os

app = Flask(__name__)
SECRET_KEY = "hardcoded_secret_123"  # VULN: Hardcoded secret
DB_PATH = "users.db"

def get_db():
    return sqlite3.connect(DB_PATH)

@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']

    # VULN: SQL Injection
    db = get_db()
    query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"
    user = db.execute(query).fetchone()

    if user:
        return f"Welcome {username}"  # VULN: XSS
    return "Invalid credentials"

@app.route('/register', methods=['POST'])
def register():
    username = request.form['username']
    password = request.form['password']

    # VULN: Weak hashing (MD5, no salt)
    hashed = hashlib.md5(password.encode()).hexdigest()

    db = get_db()
    # VULN: SQL Injection again
    db.execute(f"INSERT INTO users (username, password) VALUES ('{username}', '{hashed}')")
    db.commit()
    return "Registered!"

@app.route('/profile/<username>')
def profile(username):
    # VULN: Template injection
    template = f"<h1>Profile: {username}</h1>"
    return render_template_string(template)

@app.route('/files')
def get_file():
    filename = request.args.get('name')
    # VULN: Path traversal
    with open(f"/uploads/{filename}") as f:
        return f.read()

if __name__ == '__main__':
    app.run(debug=True)  # VULN: Debug mode in production

📝 Step-by-Step Instructions

1Security Audit with Copilot

Select the entire file and ask Copilot Chat:

Copilot Chat Prompt
Review this Flask application for security vulnerabilities. For each vulnerability found:
1. Identify the type (OWASP Top 10 category)
2. Explain the risk
3. Show the vulnerable code
4. Provide the fixed code
5. Rate severity: Critical / High / Medium / Low
✅ Expected Findings
Copilot should identify at least these 7 vulnerabilities:
  1. SQL Injection (Critical) — String formatting in SQL queries
  2. Hardcoded secrets (High) — SECRET_KEY in source code
  3. Weak password hashing (High) — MD5 without salt
  4. XSS (High) — Unescaped user input in HTML
  5. Template injection (Critical) — User input in render_template_string
  6. Path traversal (Critical) — Unsanitized file path
  7. Debug mode (Medium) — app.run(debug=True)
2Fix All Vulnerabilities

Ask Copilot to fix the entire file:

Copilot Chat Prompt
Fix all the security vulnerabilities in this file. Use:
- Parameterized queries for SQL
- bcrypt for password hashing
- Environment variables for secrets
- Input validation and sanitization
- Proper template rendering (not render_template_string)
- Path validation with os.path.realpath
- Debug mode from environment variable
3Create Organization Policy

Ask Copilot Chat to generate a Copilot usage policy:

Copilot Chat Prompt
Create a comprehensive "GitHub Copilot Usage Policy" document (Markdown) for
our engineering organization. Include sections on:
1. Acceptable Use - what Copilot can/cannot be used for
2. Code Review Requirements for AI-generated code
3. Security Guidelines - what never to share with Copilot
4. Data Privacy - handling PII and sensitive data
5. Intellectual Property - licensing considerations
6. Quality Standards - minimum review requirements
7. Prohibited Actions - things developers must never do
8. Incident Response - what to do if Copilot suggests harmful code
9. Training Requirements - onboarding for new team members
10. Metrics & Reporting - how we measure Copilot ROI

Save as COPILOT_POLICY.md

4Review AI-Generated Code Quality

Generate some code with Copilot, then critically review it. Ask Copilot to generate a user authentication middleware, then ask it to review its own output for issues:

Review prompt
Review this code you just generated. Check for:
- Security vulnerabilities
- Error handling gaps
- Edge cases not covered
- Performance issues
- Any assumptions that could be wrong
Be critical and thorough.
