Lab 3: Chat-Driven Development

Build a complete Node.js REST API for a bookstore entirely through conversations with Copilot Chat. You'll use the chat panel, inline chat, and chat participants to create routes, models, and middleware.

🎯 Objectives

  • Use Copilot Chat to scaffold a Node.js project
  • Generate Express routes through conversation
  • Use @workspace to understand project structure
  • Use inline chat (Ctrl+I) for targeted edits
  • Build a complete CRUD API without writing code manually

📋 Prerequisites

  • Node.js 18+ installed
  • Completed Labs 1-2
  • Basic understanding of REST APIs and Express.js

📝 Step-by-Step Instructions

1Initialize the Project with Chat
  1. Create a folder called bookstore-api and open it in VS Code
  2. Open Copilot Chat (Ctrl+Shift+I)
  3. Type this prompt in the chat:
Copilot Chat Prompt 
I want to create a Node.js REST API for a bookstore. Set up the project with:
- Express.js for the server
- A simple in-memory array for data storage (no database needed)
- CORS middleware
- JSON body parsing
- A Book model with: id, title, author, isbn, price, publishedYear, genre
- Port 3000

Please create a package.json and the main server file (server.js).

Copilot will generate both files. Click "Create File" or "Insert into File" for each code block.

💡
Tip: The more specific your prompt, the better the output. Notice we specified the exact model fields, port, and that we want in-memory storage.
2Generate CRUD Routes

Continue the conversation in the same chat session:

Copilot Chat Prompt 
Now create a routes file (routes/books.js) with full CRUD operations:
- GET /api/books - List all books (support ?genre and ?author query filters)
- GET /api/books/:id - Get a single book by ID
- POST /api/books - Create a new book (validate required fields)
- PUT /api/books/:id - Update a book
- DELETE /api/books/:id - Delete a book

Include proper HTTP status codes, error handling for not-found cases,
and input validation. Also add 5 sample books as seed data.
✅ Expected Output
A complete Express router with all five endpoints, proper status codes (200, 201, 404, 400), query filtering logic, and an array of 5 sample book objects.
3Add Search with Inline Chat
  1. Open routes/books.js
  2. Place your cursor after the GET /api/books route
  3. Press Ctrl+I to open inline chat
  4. Type: "Add a GET /api/books/search endpoint that searches by title or author using a ?q query parameter. Case-insensitive partial matching."
  5. Review the diff and click Accept
Expected generated route 
// GET /api/books/search?q=tolkien
router.get('/search', (req, res) => {
  const { q } = req.query;
  if (!q) {
    return res.status(400).json({ error: 'Query parameter q is required' });
  }
  const query = q.toLowerCase();
  const results = books.filter(book =>
    book.title.toLowerCase().includes(query) ||
    book.author.toLowerCase().includes(query)
  );
  res.json(results);
});
4Ask @workspace About the Project

Try these @workspace queries in Copilot Chat:

Chat Prompts
@workspace What is the overall structure of this project?

@workspace What endpoints are available in this API?

@workspace How is error handling done across the routes?

The @workspace participant scans your entire project to provide comprehensive answers.

5Add Middleware with Chat
Copilot Chat Prompt
Create a middleware file (middleware/logger.js) that logs each request with:
- Timestamp
- HTTP method
- URL path
- Response status code
- Response time in milliseconds
Format: [2024-01-15T10:30:00Z] GET /api/books 200 15ms

Also create a middleware/validator.js with a validateBook middleware
that checks all required fields exist when creating/updating a book.

After Copilot generates the files, ask it to update server.js to use the new middleware.

6Test the API
  1. Open the terminal and run npm install, then node server.js
  2. Ask Copilot Chat: "Generate curl commands to test all the endpoints in this API"
  3. Copilot will produce curl commands for each route — run them in a new terminal
Expected test commands
# List all books
curl http://localhost:3000/api/books

# Get a specific book
curl http://localhost:3000/api/books/1

# Create a new book
curl -X POST http://localhost:3000/api/books \
  -H "Content-Type: application/json" \
  -d '{"title":"Dune","author":"Frank Herbert","isbn":"978-0441013593","price":9.99,"publishedYear":1965,"genre":"Science Fiction"}'

# Search books
curl "http://localhost:3000/api/books/search?q=tolkien"

# Delete a book
curl -X DELETE http://localhost:3000/api/books/1
