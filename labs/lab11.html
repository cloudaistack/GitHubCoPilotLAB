Lab 11: Copilot in the Command Line

Master GitHub Copilot CLI to translate natural language into shell commands, explain complex commands, and supercharge your terminal workflow.

🎯 Objectives

  • Install and configure the GitHub Copilot CLI extension
  • Use gh copilot suggest to generate shell commands from plain English
  • Use gh copilot explain to understand complex commands
  • Build practical workflows with AI-generated terminal commands

📋 Prerequisites

  • GitHub CLI (gh) installed and authenticated
  • An active GitHub Copilot subscription
  • A terminal (PowerShell, Bash, or Zsh)

🚀 Steps

1 Install the Copilot CLI Extension

Open your terminal and install the Copilot extension for GitHub CLI:

Terminal
gh extension install github/gh-copilot

Verify the installation:

Terminal
gh copilot --help
✅ Expected Output
Your AI command line copilot.

Usage:
  copilot [command]

Available Commands:
  explain     Explain a command
  suggest     Suggest a command
2 Generate Commands with gh copilot suggest

The suggest command translates natural language into shell commands. It supports three command types:

  • generic shell command — general OS commands
  • gh command — GitHub CLI-specific commands
  • git command — Git version control commands

Try these exercises one at a time:

Exercise 2a: Find Large Files

Terminal
gh copilot suggest "Find all files larger than 100MB in the current directory recursively"
✅ Expected Suggestion
find . -type f -size +100M

Exercise 2b: Disk Usage Analysis

Terminal
gh copilot suggest "Show disk usage of all subdirectories sorted by size in human readable format"
✅ Expected Suggestion
du -h --max-depth=1 | sort -hr

Exercise 2c: Kill a Process on a Port

Terminal
gh copilot suggest "Find and kill the process running on port 3000"
✅ Expected Suggestion (Linux/macOS)
lsof -ti:3000 | xargs kill -9

Exercise 2d: Archive a Directory

Terminal
gh copilot suggest "Create a tar.gz archive of the src directory excluding node_modules"
✅ Expected Suggestion
tar -czf src.tar.gz --exclude='node_modules' src/

Exercise 2e: Git History Visualization

Terminal
gh copilot suggest "Show git log with a graph view, one line per commit, for the last 20 commits"
✅ Expected Suggestion
git log --oneline --graph --decorate -n 20
3 Understand Commands with gh copilot explain

The explain command breaks down complex commands into plain English. Try explaining these real-world commands:

Exercise 3a: Explain a find + grep pipeline

Terminal
gh copilot explain "find . -name '*.js' -not -path './node_modules/*' | xargs grep -l 'TODO'"
✅ Expected Explanation
This command searches for JavaScript files containing TODO comments:
• find . -name '*.js'  →  Find all files ending in .js starting from current directory
• -not -path './node_modules/*'  →  Exclude the node_modules directory
• | xargs grep -l 'TODO'  →  Pipe the filenames to grep, which lists only files containing "TODO"

Exercise 3b: Explain an awk command

Terminal
gh copilot explain "awk -F',' '{sum+=$3; count++} END {printf \"Average: %.2f\n\", sum/count}' data.csv"
✅ Expected Explanation
This command calculates the average of the 3rd column in a CSV file:
• -F','  →  Use comma as the field separator
• {sum+=$3; count++}  →  For each row, add the 3rd field to sum and increment counter
• END {printf "Average: %.2f\n", sum/count}  →  After processing all rows, print the average

Exercise 3c: Explain a curl + jq pipeline

Terminal
gh copilot explain "curl -s https://api.github.com/users/octocat/repos | jq '[.[] | {name: .name, stars: .stargazers_count}] | sort_by(.stars) | reverse | .[0:5]'"
✅ Expected Explanation
This command fetches GitHub repositories and shows the top 5 by stars:
• curl -s  →  Silently fetch the JSON response from GitHub's API
• jq '[.[] | {name, stars}]'  →  Extract name and star count from each repo
• sort_by(.stars) | reverse  →  Sort by stars in descending order
• .[0:5]  →  Take only the first 5 results
4 Practical Workflow: System Monitoring Script

Use Copilot CLI to build a monitoring script step by step. Ask for each command individually:

Terminal — Ask each one
# Ask Copilot for each of these commands:
gh copilot suggest "Show current CPU usage percentage"
gh copilot suggest "Show memory usage in human readable format"
gh copilot suggest "List the top 5 processes by memory usage"
gh copilot suggest "Show all listening network ports"
gh copilot suggest "Check if a website is reachable and show response time"

Combine the suggestions into a monitoring script:

monitor.sh — Assembled from Copilot suggestions
#!/bin/bash
# System Monitor Script — built with GitHub Copilot CLI

echo "===== System Monitor Report ====="
echo "Date: $(date)"
echo ""

echo "--- CPU Usage ---"
top -bn1 | grep "Cpu(s)" | awk '{print "Usage: " $2 "%"}'
echo ""

echo "--- Memory Usage ---"
free -h | awk '/^Mem:/ {print "Total: "$2, "Used: "$3, "Free: "$4}'
echo ""

echo "--- Top 5 Processes by Memory ---"
ps aux --sort=-%mem | head -6
echo ""

echo "--- Listening Ports ---"
ss -tlnp 2>/dev/null || netstat -tlnp 2>/dev/null
echo ""

echo "--- Website Health Check ---"
for site in google.com github.com; do
  response_time=$(curl -o /dev/null -s -w "%{time_total}" https://$site)
  echo "$site: ${response_time}s"
done

echo ""
echo "===== Report Complete ====="
💡 Tip

When Copilot suggests a command, you can choose to Run, Copy, or Revise it. Always review commands before running them, especially those with rm, kill, or sudo.

5 GitHub CLI Specific Suggestions

Copilot also excels at generating gh commands for GitHub operations:

Terminal

# List open issues with a label
gh copilot suggest "List all open issues labeled as 'bug' in this repo"

# View workflow runs
gh copilot suggest "Show the last 5 GitHub Actions workflow runs and their status"

# Clone and setup
gh copilot suggest "Fork a repo and clone my fork locally"
